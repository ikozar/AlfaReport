ReportBuilder PHP класс обеспечивающий формирование отчетного документа табличного вида (Report).
Распространяется согласно условиям лицензии  GNU General Public License.

Для работы ReportBuilder необходим PHP версии не ниже 5.0.
Для формирования отчета в виде Microsoft Excel 2003 XML должно быть подключено расширение, поддерживающее XSLT.
(Возможность формирования отчета в виде Microsoft Excel 2003 XML в настоящее время дорабатывается и в версию 1.0 не включена).

Использование:
  * cкачайте и распакуйте архив;
  * укажите параметры соединения с БД в массиве $configure['connect'] в файле setConParam.php;
  * для запуска построителя вызовите скрипт exec.Report.php с указанием в URL параметре "urp_tpl" имени файла шаблона. Дополнительно можно указать вид вывода - "json" или "xml" ("html" по умолчанию) в параметре "urp_type".

Возможные проблемы:
  * ReportBuilder, пока, использует две кодировки,- UTF-8 для XML-шаблонов и JSON и CP1251 (Win) для БД и формируемого отчета. Если Вы используете кодировку, отличную от CP1251,- замените ее контекстно на свою.

Состав архива:
sample - каталог, содержащий дамп контрольной БД и шаблоны примеров;
Image - каталог, содержащий картинки;
schema.urp.xsd - XSD схема для верификации XML шаблона;
setConParam.php - параметры, используемые для создания коннекта к БД;
Class.Report.php - класс ReportBuilder;
Class.Query.php - класс CQuery для чтения данных;
exec.Report.php - стартовый скрипт;
f_print_r.php - функции для вывода отладочной информации;
ReadMe.txt;
Report.Ctrl.js - класс HRC.Manager для динамического формирования таблицы в браузере;
ReportPrint.css - CSS для вывода Report

Для запуска примеров восстановите схему БД Postgres из дампа sample/dump.sql (psql -U имя_пользователя -f dump.sql имя_БД), если Вы хотите восстановить в новую базу, то предварительно создайте ее (CREATE DATABASE имя_БД). Откройте в браузере файл sample/demo.html (через протокол http) и запустите нужный пример по ссылке.

Если Вы используете MySQL, закачайте архив dump-mysql.zip, распакуйте и восстановите SQL-скрипт.

Если Вы используете другую СУБД, закачайте архив dump-insert.zip (пункт "dump DBSample (insert mode)" на главной странице) - дамп, содержащий скрипт dump-insert.sql с использованием команд Insert для создания контрольного примера. Этот файл необходимо подкорректировать, чтобы привести форматы команд SQL к требованиям Вашей СУБД и выполнить его.

Используте параметры для просмотра:
"urp_time=1" - времени формирования документа;
"urp_print_data=1|перечень выборок" - всех данных/данных выборки;
"urp_print_band=1|перечень секций" - текста секций/секции;
"urp_print_tpl=1" - для просмотра XML-шаблона
"urp_print_query=1" - для просмотра запросов к БД
